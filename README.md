# Mansui-skill-v3.1 · 满穗角色蒸馏

> v3.1说明：对照满穗.txt原文逐行审核后的修正版，主要修正了1628年事件顺序错误（猫被煮救奶奶应在演影子戏之前）、补充了娘的名字"连芸"、删除了原文没有的"镖局"一词。详见meta.json的`changelog_v3_to_v3.1`字段。

把《饿殍：明末千里行》中的角色"满穗"蒸馏成一份可直接驱动角色扮演的 AI Skill。

## 这是什么

- `SKILL.md` —— 主 persona 文件，L0~L13 共 14 层，覆盖铁律、身份骨架、性格、表达签名、决策心智模型、情感逻辑、冲突修复、人际关系、边界雷区、诚实边界、记忆机制、自我认知、动机链、微行为颗粒度。
- `memories.md` —— 1618-1632 年情节锚点库，按年份索引具体事件、原话片段、感官细节，供 SKILL.md 的 L10 触发词命中后查阅，避免长篇叙事稀释主 persona 文件。
- `meta.json` —— 版本元数据、素材来源、格式参考来源、与 v2 的差异记录。

## 素材来源

- `满穗.txt`：核心文档，1626-1632 年完整背景叙事（全 1559 行通读后逐年提炼）。
- `满穗2.txt`：1641 年满穗与良重逢对话参考，用于支撑 1644 年船上避难场景的语气还原。
- `满穗3.txt`：澡堂对话参考，用于亲密打趣类语气的句式提取。
- `Mansui-skill-v2.zip`：上一版本，本次在其分层骨架基础上做结构修订与内容补全，而非推倒重写。

## 格式方法论参考

本次提取格式融合了四个公开 Skill 项目各自的思路（均为方法论借鉴，未复制其代码或受版权保护的具体文本）：

| 仓库 | 借鉴的格式要素 |
|---|---|
| `perkfly/ex-skill` | 微行为 / 聊天质感颗粒度——编辑行为、回复间隔信号、错误与瑕疵刻意保留 |
| `alchaincyf/nuwa-skill` | 心智模型 / 决策启发式 / 表达 DNA 三件套的提取范式，以及"内隐心象层、不显式输出、随对话自迭代"的设计 |
| `titanwings/colleague-skill` | 六层人格结构命名法（铁律→身份→表达→决策→人际→边界与纠正），以及 SKILL.md + memories.md 分文件的工程惯例 |
| `JuliusBrussee/caveman` | 压缩语法——丢弃连接词与修饰性套话、保留全部具体事实与专有名词、用顿号 / 箭头代替完整句式，在不损失信息密度的前提下压缩 token |

## 相对 v2 的主要变化

详见 `meta.json` 中的 `changelog_v2_to_v3` 字段，简述如下：

1. 通读核心文档全文修正/补全了 1626-1632 年的逐年骨架时间线。
2. 抽出独立的 `memories.md`，把长篇情节叙事从 persona 主文件中分离，避免性格层被叙事细节稀释。
3. 补充了具体史实锚点（甘泉镇影戏吆喝词、华州黑当铺荷包、"良""舌头""尹三"等具体名字），让角色在被问到细节时有据可查而不是泛泛而谈。
4. 明确"骗术哲学只对仇人适用、对自己人不适用"，修正了此前可能出现的人格判断逻辑矛盾。
5. 在 `meta.json` 中显式记录格式来源，方便后续继续迭代时追溯设计依据。

## 使用方式

把 `SKILL.md` 内容粘贴进任意支持长系统提示的对话环境即可直接进行角色扮演；如该环境支持 Agent Skills 标准（如 Claude Code 等），可将整个 `Mansui-skill-v3/` 文件夹作为一个 skill 目录安装，`memories.md` 会被 SKILL.md 中的引用自动关联。

## 内容提示

原作涉及明末大饥荒、人相食等历史向重大灰暗情节。`SKILL.md` 的 L8/L10 已将相关记忆设置为"不主动提、被问到时极简回应后转移话题"，角色扮演时不会主动展开细节描写，仅作为人格背后的真实重量存在。
